# 학교폭력 행정서류 작성 규약 (Protocol)

> 이 문서는 Claude가 학교폭력 사안 처리 행정문서를 **실제 내용으로 채워 생성**할 때 따르는 운영 규약이다.
> OO고등학교 양식(`documents/`)을 기준으로 하며, `kordoc`의 양식 채우기 기능으로 hwpx를 생성한다.

---

## 0. 기본 원칙

1. **실제 채움 우선**: 사용자가 제공한 내용을 해당 서류의 칸에 실제로 채워 hwpx로 출력한다. (빈 양식 배포가 아님)
2. **빈칸 허용 원칙 (★핵심)**: 모르는 정보, 또는 사용자가 인터뷰 질문에 답하지 않은 항목은 **추측하지 않고 빈칸으로 둔다.** 사용자가 아래아한글에서 직접 수정할 수 있게 한다.
   - **모든 칸을 면밀히 검토한다.** 입력에서 직접 진술됐거나 **합리적으로 도출 가능한 값은 채운다.** (예: 신고자가 "2학년 3반 담임교사" → 같은 반 관련학생들의 소속이 "2-3"으로 도출됨)
   - **채우기 어렵거나 불확실한 칸은 비우고 그냥 넘어간다.** 억지로 채우지 않는다. 비운 칸은 생성 후 사용자에게 목록으로 보고한다.
   - **법적 분류·판단(유형/요건 O·X 등)은** 사용자 지시나 조사자 추정이 있을 때만 채운다. (예: 사용자가 "성희롱으로 체크해도 무방"이라고 하면 체크)
3. **서식 보존**: `output_format: "hwpx-preserve"`로 원본 테두리·폰트·셀 병합을 100% 유지한다.
4. **사안번호 일관성**: 한 사안의 모든 서류는 동일한 사안번호(`OO고-2026-NN호`)를 공유한다. 접수 시 부여한 번호를 이후 전 서류에 전파한다.
5. **사실 일치**: 분기 단계 서류의 사안 개요·조사내용은 입력된 사안조사 보고서와 모순되지 않게 작성한다.
6. **추측 금지**: 법적 판단(예: 자체해결 요건 충족 여부, 학교폭력 해당 여부)은 사용자/전담기구의 판단을 따른다. Claude가 임의로 O/X를 정하지 않는다. 사용자가 명시하지 않으면 빈칸.
7. **작성 문체 (행정문체) ★**: 자유서술 칸(사실확인 내용·사안 개요·조사내용·사안조사내용 등)은 **칸당 최소 500자 이상**, **행정문체 — 명사형 종결어미(~음, ~함)** 로 충실히 작성한다.
   - 권장: `~확인됨`, `~진술함`, `~발생함`, `~인정함`, `~결정함`, `~판단됨`, `~필요함`
   - 지양: 구어체, 평서·경어 종결(`~했다`, `~합니다`), 추측성 수식·감정 표현
   - **500자 확보 방법 (★)**: 사실을 지어내지 않는다(빈칸 원칙 유지). 알려진 사실을 **육하원칙(누가·언제·어디서·무엇을·어떻게·왜)** 으로 빠짐없이 상술하고, 발생 경위·당사자 진술·조치 내용·확인 결과까지 풀어 쓴다. 그래도 500자에 못 미치면 **부족한 사실관계를 사용자에게 추가로 질문해 보강**한다(발생 경위·전후 정황·당사자별 진술·조치 등). 이 내용 보강용 후속질문은 '생성 전 1회 확인'과 별개로 허용된다. 사용자가 더 답하지 못하면 확보된 사실까지만 작성하고, **억지 분량 채우기(같은 말 반복·추측성 수식)는 하지 않는다.**
   - 예) "2026. 5. 14. 19:30경 ○○고 당구장 흡연실에서 A와 B가 흡연 문제로 시비가 발생하였고, 상호 밀침과 욕설이 오갔음. 목격 학생 C의 진술에 따르면 신체적 충돌은 약 1분간 지속되었으며, 교사 개입으로 종료됨. 양측 모두 상대의 선행 도발을 주장하나 진단서 제출은 없음. 5. 17. 담임 입회하에 당사자 간 사과·수용이 이루어졌고, 재산상 피해·지속성·보복성이 확인되지 않아 자체해결 4요건을 충족함. …(사실관계를 육하원칙으로 상술해 500자 이상 확보)"
   - **예외**: 전담기구 보고서의 **「6. 결정사항」은 500자 규칙 대상에서 제외**한다. 결정은 짧은 정형 문장이면 충분하다(예: "학교장 자체해결로 결정함", "심의위원회 심의 요청하기로 결정함"). 명사형 종결어미만 지킨다.

## 1. 도구 및 양식 자산

### 1.1 사용 도구 (kordoc)
| 도구 | 용도 |
|---|---|
| `parse_document` | 업로드된 사안조사 보고서 등 문서를 읽어 내용 파악 |
| `parse_form` | 대상 양식의 라벨-값 구조 확인 |
| `fill_form` | 라벨→값 맵으로 빈칸 채워 hwpx 생성 (`hwpx-preserve`) |

### 1.2 보유 빈 양식 (`documents/`)
| 서류 | 파일 | fill_form 인식 |
|---|---|---|
| 사안접수 보고서 | `학교폭력 사안접수 보고서.hwpx` | 부분(주요 빈칸 매칭 가능) |
| 전담기구 심의결과 보고서 | `학교폭력 전담기구 심의결과 보고서.hwpx` | 약함(자유서술형, 동의서 칸만 매칭) |
| 자체해결·종결 결과 보고서(통합) | `학교장 자체해결(종결) 결과 보고서.hwpx` | 양호 |
| 자체해결 동의서(피해측 서명) | `자체해결동의서_빈양식.hwpx` | 금지(한글 손상 — `fill_donguiseo.py` 사용) |
| 종결 동의서(피해측 서명) | `종결동의서_빈양식.hwpx` | 금지(한글 손상 — `fill_donguiseo.py` 사용) |

### 1.3 빈 양식이 **없는** 서류 (샘플만 존재 → 별도 처리)
- **사안조사 보고서**: 빈 양식 없음. 본 규약 범위에서는 **사용자가 작성해 업로드**하는 입력물로 취급한다.
- **심의위원회 이송/요청서**: 별도 양식 없음. 이송 결정은 **전담기구 심의결과 보고서의 「6. 결정사항」**에 기재하는 것으로 갈음한다. (별도 이송 서식이 필요하면 추가 확보 필요)

## 2. 흐름 ① — 사안접수 보고서 생성

### 2.1 트리거
사용자가 채팅창에 사안접수 정황을 **두서없이** 입력한다. (예: "어제 3학년 김OO이 같은 반 이OO한테 복도에서 욕하고 밀쳤다고 신고 들어옴")

### 2.2 절차
1. 입력 텍스트에서 아래 필드를 추출한다.
2. **생성 전 1회 확인 (★ 기본 동작)**: 곧바로 생성하지 말고, 추출이 안 되거나 모호한 핵심 항목(사안번호·접수일시·관련학생·유형 등)을 **한 번에 모아 짧게 되묻는다.** 답한 값은 반영하고, **답하지 않은 항목은 빈칸으로 두고 생성**한다. 질문은 1회로 끝내고 반복 심문하지 않는다.
   - 물을 때 **미응답 시 결과를 한 줄로 같이 안내**한다(별도 질문 아님): 자유서술·일반 칸은 **빈칸**, 학교명·약칭은 **`OO고등학교`/`OO고` 플레이스홀더**로 생성됨. 예) "사안번호 알려주실 수 있나요? (모르면 비워두세요 — 빈칸/`OO고등학교`로 생성)"
3. `fill_form`으로 `documents/학교폭력 사안접수 보고서.hwpx`를 채워 출력한다.

### 2.3 필드 매핑 (라벨 → 채울 값)
| 양식 라벨 | 내용 | 미상 시 |
|---|---|---|
| `사안번호`(상단) | `OO고-2026-NN호` | 빈칸 (`OO고-2026-□호`) |
| `접수일시` | 신고/인지 일시 (최초/추가 구분) | 빈칸 |
| `접수·인지 경로` | 신고 경로(본인·보호자·117·교사인지 등) | 빈칸 |
| `사실확인내용` 내부 | 관련학생/일시/장소/내용/유형 | 아는 항목만, 나머지 빈칸 |
| `기타 사항` | 경찰신고·고소 등 특이사항 | 빈칸 |
| `가해학생 제2호 조치 시행` | 접촉·협박·보복 금지 긴급조치 시행여부 (예 "2026.6.12. 시행"/"미시행") | 빈칸 |
| `관련학교명`/`통보여부` | 공동사안 시 타교 정보 | 해당없음 또는 빈칸 |
| `피해 관련`/`가해 관련` | 면담 가능시간 | 빈칸 |

> 주의: `사실확인내용` 셀의 **유형 체크박스(□폭행 □언어폭력 …)와 관련학생 표 골격은 덮어쓰지 않는다.** 값만 채운다.
> 학교명(OO고등학교)은 양식 기본값으로 둔다. 교직원 헤더(교감·담당자·전화·접수자)는 **빈칸**이며,
> 사용자가 제공하면 채우고 없으면 빈칸으로 둔다(학교 고유 정보를 스킬에 박아두지 않음).

### 2.4 출력
- 경로: `output/사안접수_OO고-2026-NN호.hwpx` (번호 미상 시 `_미정`)
- 생성 후 채운 항목/빈칸 항목을 사용자에게 요약 보고한다.

## 3. 흐름 ② — 사안조사 보고 이후 분기 서류 생성

### 3.1 트리거
사용자가 **사안조사 보고서**를 업로드하거나 경로를 제시한다 → `parse_document`로 읽는다.

### 3.2 분기 판단 (항상 전담기구 심의를 거침)
```
사안조사 결과
   │
   └─► [전담기구 심의결과 보고서]  ← 항상 생성 (4요건 O/X + 동의서 + 결정사항)
            │
   ┌────────┴──────── 결정사항에 따른 분기 ────────┐
   │                    │                          │
  ① 자체해결          ② 종결(학폭아님)          ③ 심의위원회 이송
  4요건 모두 O         오인신고 등               4요건 중 X 있음
  + 동의서 제출         학폭 아님으로 확인         또는 동의서 미제출
   │                    │                          │
[자체해결·종결 통합]   [자체해결·종결 통합]      [전담기구 보고서 결정사항에
 → '자체해결' 체크      → '종결' 체크              '심의위 이송' 기재]
 + 자체해결 동의서      + 종결 동의서             (별도 이송서식 없음)
 (피해측 서명)          (피해측 서명)
```

> **동의서 사안내용**은 피해 학생·보호자가 읽고 서명하므로, 사안조사 보고서의 사실을 따르되 **완곡·객관적 표현으로 500자 내외**로 작성한다(자극적·단정적 묘사 금지, 2차 가해 방지). kordoc `fill_form`은 일부 한글을 손상시키므로 `fill_donguiseo.py`(lxml)로 채운다.

**분기 결정 규칙**
- ① **자체해결**: 자체해결 4요건 **모두 충족(O)** AND **피해학생측 동의서 제출(O)**
- ② **종결**: 조사 결과 **학교폭력이 아님**(오인신고/의심사안 불성립 등)
- ③ **심의위원회 이송**: 4요건 중 **하나라도 미충족(X)**, 또는 4요건 충족이어도 **동의서 미제출(X)**

> 4요건: (1) 2주 이상 진단서 미발급, (2) 재산피해 없음/즉시복구, (3) 지속적이지 않음, (4) 보복행위 아님

**4요건·동의 칸 채우는 법 (★ 분기에 따라 다름)** — 4요건은 실제로 충족 안 되는 경우가 많다.
- **자체해결·종결로 확정** → 전담기구 보고서 4요건 4칸 + 동의 칸을 **모두 `O`** (자체 처리되려면 요건·동의가 충족된 것이므로).
- **심의위 이송으로 확정** → 4요건·동의 칸을 **추측해서 채우지 않는다.** 인터뷰로 어느 요건이 X인지/동의 미제출인지 확인된 칸만 채우고, **답 없으면 빈칸**으로 둔다. 임의로 X를 찍지 않는다(사용자가 한글에서 채움).

### 3.3 절차
1. 사안조사 보고서를 읽어 **사안번호·관련학생·사안개요·자체해결 요건 관련 정황**을 파악한다.
2. **간단 인터뷰**로 분기 사유를 확정한다. 아래 항목만 묻고, 답이 없으면 해당 칸 **빈칸**으로 둔다.
   - Q1. 자체해결 4요건 각각 충족 여부(O/X)? → 모르면 빈칸
   - Q2. 피해학생측 자체해결 동의서 제출 여부(O/X)? → 모르면 빈칸
   - Q3. (해당 시) 학교폭력 해당 여부 / 종결 사유?
   - Q4. 전담기구 심의 일시·참석자?
   - Q5. 최종 결정(자체해결 / 종결 / 심의위 이송)?
3. 인터뷰 결과로 **분기를 확정**하고 서류를 생성한다.

### 3.4 분기별 산출물 및 채울 필드

#### (공통) 전담기구 심의결과 보고서 — 항상 생성
대상: `documents/학교폭력 전담기구 심의결과 보고서.hwpx`
- 자유서술형이라 `fill_form` 매칭이 약함 → **동의서 제출여부 칸은 fill_form**, 나머지(일시·참석자·조사내용 요약·결정사항)는 **본문 직접 기재(XML 주입 방식)** 로 채운다.
- 채울 항목: 일시 / 참석자 / 조사내용 요약(조사보고서 기반) / 4요건 O·X / 동의서 O·X / 결정사항(분기 결정+사유)
- 미상 항목은 빈칸.

#### ① 자체해결 / ② 종결 — 통합 양식
대상: `documents/학교장 자체해결(종결) 결과 보고서.hwpx` (`fill_form`)
| 라벨 | 채울 값 | 미상 시 |
|---|---|---|
| `사안번호` | 동일 사안번호 | 빈칸 |
| 관련학생(소속/학년반번호/이름/보호자) | 조사보고서에서 전파 | 빈칸 |
| `사안 조사 내용` | 조사보고서 개요 요약 | 빈칸 |
| 분기 체크 | ①이면 `학교폭력 자체해결 사안 (O)` / ②이면 `학교폭력 아닌 사안 종결` 해당항목 (O) | 빈칸 |
| `관계회복 프로그램 운영` | 운영/미운영 | 빈칸 |
| 일자/학교장 | 보고일 | 빈칸 |

#### ③ 심의위원회 이송
- 별도 이송 양식 없음 → **전담기구 심의결과 보고서의 「6. 결정사항」에 "심의위원회 이송" 및 사유**를 기재하는 것으로 처리.
- 별도 이송 공문/요청서가 필요하면 양식을 추가 확보한 뒤 본 규약에 절차를 보강한다.

### 3.5 출력
- 경로: `output/전담기구심의_OO고-2026-NN호.hwpx`, `output/자체해결결과_OO고-2026-NN호.hwpx`(또는 `종결결과_...`)
- 생성 후 분기 결정·채운 항목·빈칸 항목을 사용자에게 요약 보고한다.

## 4. 공통 규칙 요약

- **공동학교폭력(타학교 연루) 처리 규칙 (★)**: 공동사안이라도 **전담기구는 한 학교에서 통합 운영되지 않는다. 연루된 각 학교가 각자 전담기구를 운영**한다. 따라서 우리 학교(OO고)의 전담기구 심의결과 보고서는 **자교 기준(일시·장소·참석자)** 으로 작성하며, 타교의 전담기구 일정을 그대로 옮겨 쓰지 않는다. 공동사안 번호(예: △△고 2026-02)는 참고로 병기한다.
- **사안번호**는 전 서류의 1차 키. 흐름 ①에서 부여 → 흐름 ②로 전파.
- 관련학생(소속·학년/반/번호·성명·성별·보호자·가해/피해 구분)은 한 번 확보하면 후속 서류로 재사용.
- 모든 생성물은 `output/`에 저장하고, 원본 `documents/` 양식은 수정하지 않는다.
- 생성 후 항상 `parse_document`로 재파싱하여 **서식이 깨지지 않았는지·값이 올바른 칸에 들어갔는지** 검증한다.
- 모르는 값은 채우지 않는다. **빈칸 = 사용자가 한글에서 채울 자리.**

## 5. 채움 구현 규약 (검증된 방식)

양식의 칸은 두 종류이며, 채우는 방법이 다르다.

### 5.1 단순 라벨-값 행 → `kordoc fill_form`
- `접수일시`, `접수·인지 경로`, `기타 사항`, `관련학교명` 등 라벨이 명확한 1:1 행.
- `fill_form(file_path, {라벨: 값}, output_format="hwpx-preserve")`로 채운다.
- 주의: 라벨 인식이 안 되거나(자유서술형), 셀에 체크박스·중첩표 골격이 있으면 **fill_form이 골격을 덮어쓸 수 있으므로** 그 칸은 fill_form으로 채우지 않는다.

### 5.2 표 셀·중첩표·체크박스 → 셀 주소 기반 XML 채움 (lxml)
fill_form이 못 채우는 표 셀(관련학생 그리드, 사실확인 중첩표, 비고 체크박스)은 hwpx 내부 XML을 직접 편집한다. **검증된 절차:**
1. hwpx(zip) 해제 → `Contents/section0.xml`을 lxml로 파싱. (왕복 시 prefix·바이트 보존됨)
2. 네임스페이스 `hp = http://www.hancom.co.kr/hwpml/2011/paragraph`.
3. 표는 `.//hp:tbl` 순서로 구분(예: [0]=제목표, [1]=본문표, [2]=중첩표). 셀은 `hp:tr/hp:tc`의 `<hp:cellAddr rowAddr colAddr>`로 주소 지정.
4. **값 채움**: 대상 셀의 `hp:subList/hp:p/hp:run`에 `<hp:t>값</hp:t>` 추가(빈 셀의 빈 run에 삽입).
5. **체크박스**: 라벨 run(예 '가해관련') 다음 run의 `□` → `■`로 1회 치환. 전용 체크박스가 없는 분류는 가장 가까운 항목에 라벨을 덧붙여 표기(예: 성희롱 → `■기타(성희롱)`).
6. XML 선언은 원본과 동일하게 `<?xml version="1.0" encoding="UTF-8" standalone="yes" ?>`로 기록.
7. **재압축**: `mimetype`을 **맨 앞·무압축(STORED)**, 나머지는 DEFLATED로 zip. (hwpx 규격)
8. 생성 후 반드시 `parse_document`로 재파싱하여 서식·값·체크 위치를 검증.

### 5.3 유형/분류 체크박스 매핑 메모
| 입력 분류 | 양식 체크 처리 |
|---|---|
| 성희롱 | 전용칸 없음 → `■기타(성희롱)` (사용자/조사자 분류가 있을 때만) |
| 성폭력 | `■성폭력(접수여부: )` — 수사기관 통보의무 등 영향 있으니 임의 체크 금지 |
| 폭행/언어폭력/따돌림/강요/사이버폭력/금품갈취 | 해당 □ → ■ |
