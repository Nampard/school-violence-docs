# 학교폭력 행정서류 생성 — 직접 실행 가이드 (Codex·터미널용)

Claude의 자동 스킬(`SKILL.md`) 없이도, **순수 파이썬 스크립트만으로** 학교폭력 행정 hwpx를
채워 생성할 수 있다. 코덱스(Codex)·일반 터미널 어디서든 아래 절차대로 쓰면 된다.

> 자동 트리거(채팅에서 알아서 발동)는 Claude 전용이다. 여기서는 **사람/에이전트가 data.json을 만들고
> 스크립트를 직접 호출**한다. 채움 엔진 자체에는 Claude·MCP 의존성이 전혀 없다.

---

## 0. 요구사항

```bash
python3 --version      # 3.8+ 권장
pip install lxml       # 유일한 외부 의존성
```

생성물 검증(선택)은 kordoc MCP의 `parse_document`가 있으면 쓰고, 없으면 건너뛴다(생성은 정상).

---

## 1. 공통 사용법

```bash
cd scripts
python3 <필러> <빈양식.hwpx> <data.json> <출력.hwpx>
```

세 가지 필러:

| 서류 | 스크립트 | 빈양식 |
|---|---|---|
| 사안접수 보고서 | `fill_jeopsu.py` | `../assets/templates/사안접수_빈양식.hwpx` |
| 전담기구 심의결과 보고서 | `fill_jeondamgigu.py` | `../assets/templates/전담기구심의_빈양식.hwpx` |
| 자체해결·종결 결과 보고서(통합) | `fill_result.py` | `../assets/templates/자체해결종결_빈양식.hwpx` |

> 스크립트는 `hwpx_lib.py`를 import하므로 **`scripts/` 폴더 안에서 실행**하거나 `PYTHONPATH`에 포함해야 한다.

---

## 2. 핵심 원칙 (data.json 만들 때)

- **빈칸 허용**: 모르거나 불확실한 값은 **키를 빼거나 `""`** 로 둔다 → 그 칸은 비워진 채 출력된다(아래아한글에서 직접 채움). 억지로 추측하지 않는다.
- **학교명**: `학교명`(정식, 예 `○○고등학교`)·`학교약칭`(예 `○○고`)을 주면 채우고, 없으면 양식 기본 플레이스홀더(`OO고등학교`/`OO고`) 유지.
- **법적 분류**(피해유형·4요건 O/X)는 사용자/조사자 판단이 있을 때만. 특히 `성폭력` 체크는 임의로 하지 않는다.
- **문체**: 자유서술 칸(조사내용·사안개요 등)은 **500자 내외, 명사형 종결어미(~음/~함)**. 단 전담기구 「결정사항」은 짧은 정형 문장이면 충분.
- **관련학생이 4명을 넘으면** 필러가 표 행을 자동 복제해 전원 채운다(별지·수기 추가 불필요). `학생` 배열에 인원수대로 넣기만 하면 된다.

## 3. 서류별 data.json 예시

### 3-1. 사안접수 (`fill_jeopsu.py`)

```json
{
  "사안번호": "6",
  "접수일시": "2026년 6월 11일 15:00경",
  "신고자": "2학년 3반 담임교사",
  "접수경로": "담임 상담 중 인지·신고",
  "관련학교명": "해당없음 (같은 학교)",
  "학교명": "○○고등학교",
  "학교약칭": "○○고",
  "교감": "", "교감전화": "",
  "담당자": "", "담당전화": "", "접수자": "",
  "가해학생2호조치": "미시행",
  "사실확인": {
    "관련학생": "가해 김OO(2-3), 피해 이OO(2-3)",
    "일시": "2026. 6. 11. 6교시 미술시간",
    "장소": "",
    "내용": "활동지에 부적절한 그림을 그려 피해학생이 성적 수치심을 느낌.",
    "유형": "기타(성희롱)"
  },
  "학생": [
    {"학교": "○○고등학교", "학번": "2-3", "성명": "홍길동", "성별": "남", "비고": ["가해관련"]},
    {"학교": "○○고등학교", "학번": "2-3", "성명": "김철수", "성별": "", "비고": ["피해관련", "다문화학생"]}
  ]
}
```
- 유형 키워드 예: `폭행 / 언어폭력 / 따돌림 / 강요 / 사이버폭력 / 금품갈취 / 기타 / 성희롱(→■기타(성희롱)) / 성폭력`.
- `비고` 체크 키워드: `가해관련 / 피해관련 / 학생선수 / 장애학생 / 특수교육대상자 / 다문화학생 / 탈북학생`.

```bash
python3 fill_jeopsu.py ../assets/templates/사안접수_빈양식.hwpx data.json 사안접수_출력.hwpx
```

### 3-2. 전담기구 심의결과 (`fill_jeondamgigu.py`)

```json
{
  "사안번호": "5",
  "학교명": "○○고등학교", "학교약칭": "○○고",
  "일시": "2026년 6월 15일 (월요일) 13:00",
  "참석자": "교감, 학생생활안전부장, 학부모위원 3명",
  "조사내용": "2026. 5. 14. … 상호 밀침·욕설이 있었음. … 자체해결 4요건을 충족함.",
  "요건": ["O", "O", "O", "O"],
  "동의": "O (자체해결 동의)",
  "결정": "학교장 자체해결로 결정함."
}
```
- **4요건·동의 채우는 법**: 자체해결/종결로 확정이면 `요건`·`동의` 모두 `O`. **심의위 이송이면 추측 금지** — 확인된 칸만 채우고(예 어느 요건이 `X`), 모르면 그 항목을 빼서 빈칸으로 둔다.
- `참석자`는 예시일 뿐 양식엔 박혀있지 않다. 주면 채우고 없으면 빈칸.

```bash
python3 fill_jeondamgigu.py ../assets/templates/전담기구심의_빈양식.hwpx data.json 전담기구_출력.hwpx
```

### 3-3. 자체해결·종결 결과 (`fill_result.py`)

```json
{
  "사안번호": "5",
  "학교명": "○○고등학교", "학교약칭": "○○고",
  "mode": "자체해결",
  "학생": [
    {"학교": "○○고등학교", "학년반번호": "2-3-08", "성명": "이민수", "보호자": ""}
  ],
  "사안조사내용": "… 명사형 종결어미로 500자 내외 …",
  "관계회복": "미운영",
  "오인신고항목": 1
}
```
- `mode`: `"자체해결"` 또는 `"종결"`.
- `오인신고항목`은 `mode="종결"`일 때만 의미 있음 (학폭 아님 종결 사유 1/2/3).

```bash
python3 fill_result.py ../assets/templates/자체해결종결_빈양식.hwpx data.json 결과_출력.hwpx
```

## 4. 분기 한눈에 (흐름 ②: 사안조사 보고서 이후)

전담기구 심의는 **항상** 생성하고, 결정에 따라 결과 서류가 갈린다.

| 분기 | 조건 | 생성물 |
|---|---|---|
| 자체해결 | 4요건 모두 O **그리고** 피해측 동의 O | 전담기구 보고 + 자체해결 결과(`mode:자체해결`) |
| 종결 | 학교폭력 아님(오인신고 등) | 전담기구 보고 + 종결 결과(`mode:종결`) |
| 심의위 이송 | 4요건 중 X **또는** 동의 미제출 | 전담기구 보고만(결정사항에 이송 기재) |

> 공동학폭(타학교 연루)이면 전담기구는 학교마다 각자 운영 → 우리 학교 기준(일시·참석자)으로 작성.

## 5. 생성 후 검증 (선택)

kordoc MCP가 있으면 `parse_document(출력.hwpx)`로 재파싱해 표·체크박스·값 위치를 확인한다.
없으면 아래아한글에서 직접 열어 확인하면 된다. 표 셀 주소 등 상세는 `references/field-maps.md` 참고.
